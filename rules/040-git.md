# Git Rules

## Golden Rule

One task = One commit = One feature branch

## Commit Message Format

`[type]: [short description]`

Allowed types:

- `feat:` ‚Äî New feature
- `fix:` ‚Äî Bug fix
- `test:` ‚Äî Add/change test
- `refactor:` ‚Äî Code change without behavior change
- `docs:` ‚Äî Documentation change
- `chore:` ‚Äî Maintenance tasks

## Before Commit

1. `uv run ruff check --fix .` ‚Äî Fix linting issues
2. `uv run pytest` ‚Äî All tests must pass
3. Run security scan on staged files (see Security Check below)
4. Update all relevant docs ‚Äî **before** staging the commit:
   - `README.md`, `CHANGELOG.md`, `AGENTIC-CODING-SETUP.md`
   - Any file under `docs/`, `rules/`, `skills/`, `workflows/`
   - Any other documentation file that describes the changed behaviour
   - If a doc file is no longer accurate after this change, update or note it
5. Remove temporary files created during the task: scratch files, debug logs, `*.tmp`, `_draft.*`, `*_test.*` artifacts, and any file not intended to be part of the final commit. The pre-commit hook (Rule 5) lists untracked throwaway files as a non-blocking reminder.
6. Then: `git commit -m "[type]: [description]"`

## Security Check (Mandatory Before Every Commit)

Run `git diff --cached` and verify staged files contain NONE of the following:

- Real names or surnames
- Email addresses
- Phone numbers
- API keys, tokens, or any secret strings
- Personal file paths (paths pointing to private local files)
- Hardcoded data from personal documents (CV, contracts, credentials)

If anything suspicious is found ‚Üí `git restore --staged <file>` and ask the user.
When in doubt, do NOT commit.

This manual check is backstopped by the pre-commit hook
(`templates/hooks/pre-commit`, Rule 4), which scans newly **added** lines for
secrets / personal data and blocks the commit ‚Äî but the manual review remains
mandatory; the hook is a safety net, not a replacement.

## Commit Message Privacy (Non-Negotiable)

When a commit fixes or touches a security/privacy issue, the commit message must
**never describe the issue or reveal the sensitive data**. Forbidden examples:

- `remove phone number 0775XXXXXX from config`
- `scrub email / API key from <file>`
- `remove the candidate's name from a comment`

Why: commit messages are permanent and searchable in git history. Naming the
change is a signpost pointing an attacker straight at the sensitive data ‚Äî and
the message persists even after the data itself is removed.

Rules:

- Describe **only** the neutral functional change (the *what*, not the *leak*).
- If a privacy hardening must be noted, keep it generic and location-free
  (e.g. `harden config handling`), never the specific value, field, or file.
- Applies to the subject, body, and any trailer.
- The same principle applies to hook/CI output: report the file and the *type*
  of finding, never the matched value.

## Commit Identity (Non-Negotiable)

**Author email:** Every commit must be signed with `sushiant60@gmail.com`.
Before committing, verify: `git config user.email` returns `sushiant60@gmail.com`.
If not, set it: `git config user.email "sushiant60@gmail.com"`.

**No AI co-authorship ‚Äî ever.**
No agent, assistant, or AI tool may add itself as a contributor to any commit.
Forbidden in all forms:

- `Co-Authored-By: Claude ...`
- `Co-Authored-By: GitHub Copilot ...`
- `Co-Authored-By: <any-ai>@<any-domain>`
- Any variation of AI attribution in the commit message, body, or trailer

Commit messages document the *change*, not *who or what* produced it.
The human author is the sole credited contributor.

## Merge Gate (Non-Negotiable)

**Never merge a feature branch without explicit user approval.**

After every commit on a feature branch:

1. Tell the user: "Ready to test ‚Äî please confirm before I merge."
2. Wait for the user to test and explicitly approve.
3. Only then run `git merge` + `git branch -d`.

Merging before approval removes the user's ability to reject broken changes without a revert.

## Personal / Memory Workspace Files (Non-Negotiable)

Files used only for our own notes and session memory ‚Äî `TODO.md`, `SESSION.md`,
`TASKS.md`, `ROADMAP.md`, `*.session.md`, `*.local.md` ‚Äî must **never** be
committed or pushed to GitHub, in this repo or any project repo.

- They must always be listed in `.gitignore`.
- The pre-commit hook (`templates/hooks/pre-commit`, Rule 3) blocks them at
  the git level even if someone force-adds with `git add -f`.
- If one of these files is already tracked in a repo, remove it with
  `git rm --cached <file>` and add it to `.gitignore` ‚Äî do not just edit it.

## Forbidden

CRITICAL: NEVER make any code modifications or git commits while on the 'main' branch. Your FIRST action for any task MUST BE: `git checkout -b <branch_name>`. The system will block any direct commits to main.

- Large commits with multiple scattered changes
- Commit messages like "fix", "update", "changes"
- `Co-Authored-By:` or any AI identity attribution in commits (see Commit Identity above)
- Committing with an author email other than `sushiant60@gmail.com`
- Merging a feature branch before the user has tested and approved
