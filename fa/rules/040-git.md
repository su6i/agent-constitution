# قوانین گیت

## قانون طلایی
هر تسک = یک کامیت = یک شاخه (Branch) ویژگی.

## فرمت پیام کامیت
`[نوع]: [توضیح کوتاه]`
انواع: feat, fix, test, refactor, docs, chore.

## قبل از کامیت
۱. اجرای روف (`ruff`) برای اصلاح استایل.
۲. اجرای تست‌ها (`pytest`).
۳. بررسی اسرار و کلیدهای امنیتی.

## ممنوع
CRITICAL: NEVER make any code modifications or git commits while on the 'main' branch. Your FIRST action for any task MUST BE: `git checkout -b <branch_name>`. The system will block any direct commits to main.
- پیام‌های نامفهوم مثل "update" یا "changes".
