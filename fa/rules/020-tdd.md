# قوانین TDD

## ترتیب اجباری
۱. ابتدا تستی بنویسید که شکست می‌خورد (FAIL).
۲. کمترین کد ممکن را برای پاس شدن تست بنویسید.
۳. بازنویسی (Refactor) کنید.
۴. دستور `uv run pytest` را اجرا کنید.

## قوانین تست
- نام فایل: `test_[module_name].py` در پوشه `tests/`.
- هر تست فقط یک مسئولیت دارد (Single Responsibility).
- استفاده از fixtures برای تنظیمات تکراری.

## پوشش کد (Coverage)
- حداقل ۸۰٪ پوشش برای ادغام در شاخه اصلی.
